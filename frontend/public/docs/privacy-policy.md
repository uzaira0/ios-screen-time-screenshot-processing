# Privacy Policy

**Last Updated:** November 22, 2025

## Introduction

Screenshot Processor ("we", "our", "the app") is committed to protecting your privacy. This Privacy Policy explains how your data is handled when you use our application.

**Key Principle:** We believe your data belongs to you. We've designed Screenshot Processor to give you complete control over your information.

## Two Operating Modes

Screenshot Processor operates in two distinct modes with different privacy implications:

### Local Mode (WASM) - Maximum Privacy

When you use Local Mode:

✅ **Complete Privacy**
- All processing happens in your browser
- Data NEVER leaves your device
- No server communication
- No tracking or analytics
- No data collection of any kind

✅ **Your Control**
- Data stored locally in your browser
- You control all data
- Delete anytime
- Export anytime

✅ **Zero External Dependencies**
- No server required
- No cloud storage
- No third-party services
- 100% offline capable

**Storage Location:** Browser IndexedDB on your device

**Data Deletion:** Clear browser data or use in-app "Clear All Data"

### Server Mode - Collaborative Processing

When you use Server Mode:

⚠️ **Server-Based Processing**
- Screenshots uploaded to server
- Processed on server backend
- Stored in server database
- Accessible from any device (with login)

⚠️ **Data Collection**
- Screenshots you upload
- Annotations you create
- User account information
- Processing logs
- Usage statistics (server-side)

⚠️ **Data Sharing**
- Shared with other authenticated users (if multi-user)
- Subject to server administrator's policies
- May be backed up by server operator

**Storage Location:** Server database operated by your organization

**Data Deletion:** Contact server administrator

## What Data We Process

### Local Mode

When using Local Mode, the following data is stored **only on your device**:

1. **Screenshots**
   - Image files you upload
   - Stored as browser blobs
   - Never transmitted

2. **Extracted Data**
   - Hourly usage values
   - Titles and totals
   - Grid coordinates
   - Processing metadata

3. **Annotations**
   - Your edits and corrections
   - Notes and tags
   - Timestamps

4. **Application Settings**
   - Your preferences
   - Processing mode selection
   - UI customizations

### Server Mode

When using Server Mode, the following data is sent to the server:

1. **Account Data** (if authentication enabled)
   - Username
   - Email address (optional)
   - Password (hashed)
   - Account creation date

2. **Uploaded Content**
   - Screenshot images
   - Image metadata
   - Upload timestamps

3. **Annotations**
   - Extracted data
   - Manual corrections
   - Notes and tags
   - Annotator ID

4. **Usage Data**
   - Processing requests
   - API calls
   - Session information
   - Error logs

## How We Use Your Data

### Local Mode

Your data is used solely for:
- Processing screenshots
- Displaying results
- Exporting data you request
- Improving your experience

**We do NOT:**
- Transmit your data
- Analyze your usage
- Share with third parties
- Track your behavior
- Serve advertisements

### Server Mode

Your data is used for:
- Processing screenshots
- Storing annotations
- Enabling collaboration
- Providing consensus features
- Improving service quality

**Server administrators may:**
- Access uploaded content
- Monitor system usage
- Create backups
- Analyze aggregate statistics

## Data Security

### Local Mode Security

**Browser Security:**
- Data protected by browser's same-origin policy
- IndexedDB encrypted by browser
- Cleared when browser data cleared

**Recommendations:**
- Use trusted, updated browser
- Enable browser security features
- Don't use public/shared computers for sensitive data
- Lock your device when not in use
- Export backups to secure location

### Server Mode Security

**Server-Dependent:**
- Security managed by server operator
- Check with your server administrator
- May include HTTPS, authentication, encryption

**Your Responsibilities:**
- Use strong passwords
- Enable 2FA if available
- Don't share account credentials
- Log out on shared devices

## Data Retention

### Local Mode

**You control retention:**
- Data persists until you delete it
- Browser may clear data if storage full
- Clearing browser data deletes app data

**Recommendations:**
- Export backups regularly
- Don't rely on browser storage long-term
- Clear old data you don't need

### Server Mode

**Server-Dependent:**
- Retention policies set by server administrator
- May have automatic cleanup
- May keep indefinite backups

**Check with your server for:**
- Data retention periods
- Backup policies
- Deletion procedures
